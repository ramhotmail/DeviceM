# نظام تكويد الأجهزة الطبية (Equipment Tags System)

تطبيق ويب بلغة Python (Flask) + قاعدة بيانات **Turso** (سحابية، متوافقة 100% مع SQLite، ومجانية ودائمة)، يعمل على:
- إضافة/تعديل/حذف بيانات الأجهزة الطبية.
- إنشاء **كارت** مطابق لتصميم الكارت الأصلي (بنفس الحقول والشكل)، قابل للطباعة مباشرة من المتصفح.
- توليد **QR Code** تلقائي لكل جهاز، يفتح صفحة الكارت الخاصة به عند مسحه.
- **البيانات محفوظة دائمًا** ومش بتتمسح لو السيرفر أعاد التشغيل أو عملت Redeploy.

---

## 1) اعمل قاعدة بيانات Turso مجانية (5 دقايق بس)

1. ادخل https://turso.tech وسجّل حساب مجاني (بإيميلك أو GitHub).
2. سطّب الأداة CLI الخاصة بيهم، أو ببساطة من نفس لوحة التحكم بتاعتهم على الموقع اضغط **Create Database**.
   - اختار اسم للقاعدة (مثلاً: `equipment-tags`).
   - اختار أقرب منطقة (Region) لبلدك.
3. بعد ما تتعمل القاعدة، هتلاقي في صفحتها:
   - **Database URL**: بيبدأ بـ `libsql://...turso.io`
   - **Auth Token**: اضغط **Create Token** أو **Generate Token** وانسخه (يظهر مرة واحدة بس، احفظه فورًا).

> القاعدة المجانية على Turso بتدّيك 500 قاعدة بيانات و 9 جيجا تخزين تقريبًا — أكتر من كافي لهذا الاستخدام، ومفيش مواعيد انتهاء أو حذف تلقائي للبيانات.

---

## 2) التشغيل على جهازك محليًا (اختياري، للتجربة)

```bash
pip install -r requirements.txt
export TURSO_DATABASE_URL="libsql://your-db-name-your-org.turso.io"
export TURSO_AUTH_TOKEN="التوكن-اللي-نسخته"
python app.py
```

ثم افتح المتصفح على: `http://127.0.0.1:5000`

> لو شغّلت التطبيق من غير ما تحط `TURSO_DATABASE_URL`، هيشتغل تلقائيًا بملف SQLite محلي عادي (وضع تجريبي بس، مش هيتحفظ لما ترفعه Render).

> غيّر شعار الجهة الافتراضي باستبدال الملف: `static/logo/logo.png` بشعار مستشفياتكم الحقيقي (نفس الاسم، أي مقاس مربّع تقريبًا).

---

## 3) رفعه على Render.com (استضافة مجانية) خطوة بخطوة

### الخطوة 1: ارفع الملفات على GitHub
1. اعمل حساب على https://github.com (لو مفيش عندك).
2. اعمل Repository جديد (خاص أو عام)، وارفع كل ملفات هذا المجلد بداخله.
   - أسهل طريقة: من نفس صفحة الـ Repository اضغط **Add file → Upload files** واسحب الملفات.

### الخطوة 2: اعمل حساب على Render
1. ادخل https://render.com وسجّل دخول بحساب GitHub.
2. اضغط **New +** → **Web Service**.
3. اختر الـ Repository اللي رفعته.

### الخطوة 3: إعدادات الخدمة
Render هيكتشف تلقائيًا ملف `render.yaml` المرفق، لكن لو طلب منك تدخل الإعدادات يدويًا:
- **Environment**: Python 3
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `gunicorn app:app --bind 0.0.0.0:$PORT`
- **Instance Type**: Free

### الخطوة 4: أضف متغيرات Turso (مهم جدًا)
من صفحة الخدمة على Render، روح لـ **Environment** وضيف المتغيرين دول:
| Key | Value |
|---|---|
| `TURSO_DATABASE_URL` | رابط قاعدتك من Turso (زي `libsql://equipment-tags-yourorg.turso.io`) |
| `TURSO_AUTH_TOKEN` | التوكن اللي نسخته من Turso |

احفظ (Save Changes)، هيعمل Render إعادة نشر تلقائيًا. بعد كده هيديك رابط زي:
`https://equipment-tags-app.onrender.com`

## 4) تسجيل الدخول والصلاحيات (جروب محدود)

التطبيق دلوقتي **مش مفتوح للعامة** — فيه صفحة تسجيل دخول إجبارية، ومفيش تسجيل حساب جديد بنفسك؛ الحسابات كلها بيضيفها الأدمن بس من صفحة "المستخدمين".

### أول تشغيل
أول ما التطبيق يشتغل (محليًا أو على Render)، بيتعمل تلقائيًا حساب أدمن افتراضي واحد:
- **اسم المستخدم**: `admin`
- **كلمة المرور**: `admin123`

⚠️ **مهم جدًا**: غيّر بيانات الأدمن الافتراضية دي عن طريق إضافة المتغيرين البيئيين دول على Render **قبل** أول تشغيل:
| Key | مثال |
|---|---|
| `ADMIN_USERNAME` | `super_admin` |
| `ADMIN_PASSWORD` | كلمة مرور قوية من اختيارك |

(لو التطبيق سبق واشتغل فعلاً بحساب `admin/admin123`، ادخل بيه وعدّل الحساب من صفحة المستخدمين — اعمل حساب أدمن جديد بباسورد قوي واحذف القديم.)

### الصلاحيات (جروبين بس)
- **أدمن**: يضيف/يعدل/يحذف الأجهزة، يدير المستخدمين (يضيف/يحذف حسابات)، ويشوف التقارير.
- **مشاهد**: يشوف قائمة الأجهزة وكروتها ويطبعها، ويشوف التقارير فقط — مينفعش يضيف/يعدل/يحذف ولا يدخل صفحة المستخدمين.

الأدمن بس هو اللي يقدر يضيف مستخدمين جدد من صفحة **👥 المستخدمين** في الأعلى، ويحدد كل حساب صلاحيته (أدمن/مشاهد).

---

## 5) تقرير الأجهزة حسب التاريخ

من صفحة **📊 التقرير** تقدر:
- تحدد فترة تاريخ (من - إلى) وتشوف كل الأجهزة اللي اتضافت في الفترة دي.
- تشوف عدد الأجهزة المضافة في كل يوم داخل الفترة.
- تطبع التقرير مباشرة بزرار الطباعة.

التقرير بيفلتر على أساس **تاريخ إضافة الجهاز للنظام** (created_at).

---

## ✅ التخزين دلوقتي دائم فعليًا

بعد الربط بـ Turso، بيانات الأجهزة والصور بتتحفظ في قاعدة بيانات سحابية دائمة — مش هتتمسح لو:
- عملت Redeploy أو تحديث كود جديد.
- السيرفر على Render "نام" بسبب عدم الاستخدام وبعدين اشتغل تاني.

> ملحوظة صغيرة: **صور الأجهزة المرفوعة** (ملفات الصور نفسها، مش بيانات الجهاز) لسه بتتخزن على قرص Render المؤقت مش داخل Turso (لأن Turso قاعدة بيانات مش تخزين ملفات). يعني بيانات الكارت (الاسم، الكود، الموديل...إلخ) محفوظة دائمًا، لكن لو أعدت النشر ممكن تحتاج ترفع صور الأجهزة تاني. لو حابب نخزن الصور بشكل دائم كمان، أقدر أضيفلك حل زي رفعها كـ Base64 جوه Turso نفسه أو استخدام تخزين سحابي مجاني للصور — قولّي لو محتاج كده.

## استخدام التطبيق
1. سجّل الدخول بحساب الأدمن.
2. من الصفحة الرئيسية اضغط **+ إضافة جهاز جديد**.
3. املأ البيانات وارفع صورة الجهاز (اختياري) واضغط **حفظ**.
4. هتنقلك مباشرة لصفحة **الكارت** بنفس تصميم الكارت الأصلي مع QR Code.
5. اضغط **طباعة الكارت** لطباعته مباشرة (مقاس بطاقة تعريف/ID Card).
6. من القائمة الرئيسية تقدر تبحث، تعدل، أو تحذف أي جهاز (لو أدمن).
7. من صفحة **📊 التقرير** تابع الأجهزة المضافة حسب أي فترة تاريخ.
8. من صفحة **👥 المستخدمين** (أدمن فقط) أضف حسابات جديدة وحدد صلاحياتها.
